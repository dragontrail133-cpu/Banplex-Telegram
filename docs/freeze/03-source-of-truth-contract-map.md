Freeze date: `2026-04-19`

## Aturan kontrak lintas domain

- Pola final untuk domain inti adalah `read direct or server read model`, lalu `write API/RPC only`.
- `Referensi` / `Master` adalah domain core-release fondasional yang dipakai semua form inti; kalau boundary runtime masih transitional, status domain ini tetap core release dan bukan supporting module.
- Domain sensitif yang wajib lewat API/RPC untuk write: `auth/workspace`, `Jurnal`, `Pemasukan Proyek`, `Pengeluaran`, `Dokumen Barang`, `Dana Masuk / Pinjaman`, `Tagihan`, `Pembayaran`, `Halaman Absensi`, `Catatan Absensi`, `Tagihan Upah`, dan `Reports`.
- `Riwayat` adalah completed/history surface; deleted recovery dan restore memakai `Recycle Bin` yang terpisah.
- `Halaman Absensi` existing adalah surface input absensi harian. `Catatan Absensi` adalah halaman baru untuk histori, filter, dan rekap payroll; keduanya bukan row utama ledger finance.
- Dalam konteks payroll operasional, worker adalah parent utama grouping; payroll payable yang boleh muncul di `Jurnal` adalah `Tagihan Upah` per worker.
- `Payment Receipt PDF` adalah output turunan resmi dari `Pembayaran`, boleh diregenerate, dan tidak pernah menjadi source of truth.
- `pdf_settings` adalah boundary konfigurasi untuk PDF bisnis user-facing; itu terpisah dari `Payment Receipt PDF` dan tidak mengubah settlement source of truth.
- `Telegram assistant` adalah surface percakapan read-only; boleh menyimpan session memory ringkas untuk continuation dalam bentuk `summary`, `last_turn`, `last_route`, `entity_hints`, dan hybrid transcript pendek, sementara command/inline/read text semuanya wajib melewati planner + verifier yang sama sebelum AI writer merangkai balasan natural-language.
- Area legacy atau superseded tidak boleh menjadi target task baru bila masih ada jalur final yang tercatat di bawah.

## Runtime exceptions / transitional boundaries yang masih aktif

Boundary di bawah ini nyata di runtime repo saat ini, tetapi bukan pola baru untuk domain inti:

- `src/store/useTeamStore.js` masih melakukan read/write client langsung ke `team_members` dan `invite_tokens` untuk capability `Tim`,
- `src/store/useMasterStore.js` masih memegang CRUD client langsung untuk master data dan RPC worker-related,
- `src/store/useHrStore.js` masih memegang CRUD client langsung untuk `HRD` dan `Penerima Manfaat`,
- `src/store/useFileStore.js` tetap menangani upload storage dan `file_assets` secara langsung; relation attachment parent tetap lewat `/api/records`,
- `src/components/PayrollManager.jsx` masih memanggil RPC `fn_generate_salary_bill` dari frontend untuk generate `Tagihan Upah`.

Exception di atas harus diperlakukan sebagai transitional boundary yang terdokumentasi, bukan alasan untuk memperluas direct Supabase write ke finance core.
